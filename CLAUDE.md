# CLAUDE.md — SSV Scanner

This file guides Claude Code when working with the SSV Scanner repository. Read this fully before making any changes.

## Project Overview

SSV Scanner is a **TypeScript library and CLI tool** for retrieving on-chain event data from the SSV Network smart contract. It is used to fetch:

- **Cluster snapshots** — the latest cluster state (required input for SSV contract transactions)
- **Owner nonce** — the current nonce for a given owner address
- **Operator data** — operator public keys associated with an owner

The library is published as an npm package and consumed by other SSV tooling (e.g. ssv-keys). It also ships a standalone CLI (`ssv-keys` binary).

## Build & Dev Commands

```bash
yarn                  # Install dependencies
yarn dev:cli ...      # Run CLI as TypeScript (ts-node, dev mode)
yarn cli ...          # Run compiled CLI (requires build first)
yarn lint             # ESLint check
yarn build            # Compile TypeScript → dist/tsc/
yarn esbuild          # Bundle with esbuild → dist/
yarn build-all        # clean + build + copy-json + esbuild (full release build)
yarn copy-json        # Copy ABI JSON files to dist output
```

**Pre-commit hook** (`yarn pre-commit`) runs: `yarn test && yarn lint && yarn build-all`

### Scanner Classes

All scanners extend `BaseScanner` ([src/lib/BaseScanner.ts](src/lib/BaseScanner.ts)), which validates and normalizes constructor params (`nodeUrl`, `network`, `ownerAddress`).

| Class | File | Purpose |
|---|---|---|
| `ClusterScanner` | `src/lib/ClusterScanner/ClusterScanner.ts` | Scans events to find latest cluster snapshot for a given owner + operator set |
| `NonceScanner` | `src/lib/NonceScanner/NonceScanner.ts` | Finds current owner nonce from contract events |
| `OperatorScanner` | `src/lib/OperatorScanner/OperatorScanner.ts` | Retrieves operator public keys for an owner |

### Contract Provider

[src/lib/contract.provider.ts](src/lib/contract.provider.ts) maps network strings to ABI files and contract settings:

```ts
ContractVersion = {
  MAINNET:       'prod:v4.mainnet',
  HOODI:         'prod:v4.hoodi',
  HOODI_STAGE:   'stage:v4.hoodi',
  LOCAL_TESTNET: 'local:v4.testnet',
  FUSAKA_STAGE:  'stage:v4.fusaka',
}
```

Each ABI JSON file in `src/shared/abi/` contains: `contractAddress`, `abi`, `genesisBlock`.

### Scanning Strategy

`ClusterScanner` scans backwards from the latest block in chunks:
- Starts with `MONTH` (30 days ≈ 5400 blocks × 30) chunks
- Falls back to `WEEK`, then `DAY` on failure
- Filters events by owner address (topic index 1) and operator IDs
- Relevant events: `ClusterDeposited`, `ClusterWithdrawn`, `ClusterReactivated`, `ValidatorRemoved`, `ValidatorAdded`, `ClusterLiquidated`, `ClusterBalanceUpdated`, `ClusterMigratedToETH`

### CLI Commands

Defined in `src/commands/` and registered in `src/cli.ts`:

| Command | Description |
|---|---|
| `cluster` | Fetch latest cluster snapshot |
| `nonce` | Fetch current owner nonce |
| `operator` | Fetch operator data for an owner |

### Public API (Library)

[src/main.ts](src/main.ts) exports:
- `ClusterScanner`

## ABI Files

ABI files in `src/shared/abi/` are **not auto-generated by a script in this repo** — they are updated manually when contracts are deployed. Each file must have:

```json
{
  "contractAddress": "0x...",
  "genesisBlock": 123456,
  "abi": [...]
}
```

After updating an ABI, run `yarn build-all` to ensure the JSON is copied to `dist/` via `yarn copy-json`.
